# AGENTS.md — правила развёртывания лендинга Virale

> Документ для ИИ-агента (Claude Code / Cursor / Copilot и т.п.), который будет разворачивать этот лендинг.
> Следуй инструкциям буквально. Это **статический сайт** — не превращай его в сборочный проект, не добавляй фреймворки, не «улучшай» структуру.

## 1. Контекст
- Это готовый маркетинговый лендинг «Virale».
- Технологии: **чистый HTML + CSS + ванильный JS в одном файле** `index.html`. Сборки нет, бэкенда нет, базы данных нет, package.json нет.
- Цель: отдать содержимое этой папки статическим веб-сервером по URL `https://chatplace.io/ru/virale/`.

## 2. Манифест файлов (что должно поехать на сервер)
```
index.html                ← точка входа (вся страница: HTML+CSS+JS)
README.md                 ← инструкция для людей (на сервер можно не заливать)
AGENTS.md                 ← этот файл (на сервер можно не заливать)
fonts/                    ← 5 файлов .ttf (Pragmatica Extended, Roboto) — ОБЯЗАТЕЛЬНО
*.jpg / *.webp / *.png    ← изображения (обложки, фото, логотипы)
*.mp4                     ← демо-видео (основной вес, ~28 МБ суммарно)
```
Всего ~60 ассетов. Все ссылки в `index.html` **относительные** (например `src="app-demo.mp4"`, `url('fonts/Roboto-Regular.ia0dPpEo.ttf')`).

## 3. ЖЁСТКИЕ ОГРАНИЧЕНИЯ (не нарушать)
- НЕ переименовывай `index.html` и ассеты — имена файлов захардкожены в коде.
- НЕ меняй структуру папок: `fonts/` должна лежать рядом с `index.html`.
- НЕ трогай относительные пути внутри `index.html`.
- НЕ запускай минификацию/бандлинг/транспиляцию — файл уже финальный.
- НЕ добавляй Node/PHP/build-шаги. Любой такой шаг — ошибка.
- Заливай **всю папку целиком**, включая `fonts/` и все медиа.

## 4. Шаги развёртывания

### Если сервер на nginx
1. Скопировать содержимое папки в каталог на сервере, напр. `/var/www/virale-landing/`.
2. Добавить location-блок:
```nginx
location /ru/virale/ {
    alias /var/www/virale-landing/;
    index index.html;
    try_files $uri $uri/ /ru/virale/index.html;
}
```
3. Проверить конфиг и перезагрузить: `nginx -t && systemctl reload nginx`.

### Если Apache
Положить папку в DocumentRoot по пути `/ru/virale/`. `index.html` отдаётся по умолчанию. `.htaccess` не требуется.

### Если S3 / CDN / Cloudflare Pages / Netlify / GitHub Pages
Загрузить папку как статику, корневой документ — `index.html`. Сохранить вложенную папку `fonts/`.

## 5. MIME-типы (проверить, обычно по умолчанию верны)
- `.mp4` → `video/mp4`
- `.webp` → `image/webp`
- `.ttf` → `font/ttf`
- `.png` `.jpg` → `image/png` / `image/jpeg`
Если видео не проигрывается или шрифты не грузятся — почти всегда причина в неверном MIME или потерянной папке `fonts/`.

## 6. Внешние зависимости (НЕ входят в папку, грузятся с CDN)
- Иконки Tabler: `https://cdn.jsdelivr.net/npm/@tabler/icons-webfont@3/dist/tabler-icons.min.css` (подключено в `<head>`). Работает из коробки.
  - Опционально для автономности: скачать этот CSS и шрифты пакета `@tabler/icons-webfont@3`, положить локально и заменить ссылку в `<head>` на относительную. Это единственная допустимая правка кода.
- `og:image` указывает на `https://storage.googleapis.com/chatplace-website/meta/ru/virale.png` (инфраструктура заказчика). Менять только по запросу заказчика.

## 7. Проверка после деплоя (критерии приёмки)
Выполнить и убедиться, что всё `200`:
```bash
curl -s -o /dev/null -w "%{http_code}\n" https://chatplace.io/ru/virale/
curl -s -o /dev/null -w "%{http_code}\n" https://chatplace.io/ru/virale/fonts/Roboto-Regular.ia0dPpEo.ttf
curl -s -o /dev/null -w "%{http_code}\n" https://chatplace.io/ru/virale/app-demo.mp4
```
Затем открыть страницу в браузере и проверить:
- [ ] Страница открывается, заголовки набраны фирменным шрифтом (Pragmatica Extended), а не системным.
- [ ] В консоли браузера (DevTools → Console/Network) **нет ошибок 404** на ассеты.
- [ ] Демо-видео (в макете телефона и в блоке видеостудии) проигрываются и зациклены.
- [ ] На мобильном (ширина ~390px): нет горизонтальной прокрутки, блоки листаются без рывков, картинки в блоке «Кому подойдёт» не наезжают на текст.
- [ ] og-превью: при вставке ссылки в мессенджер подтягивается картинка и описание.

Развёртывание считается успешным, когда все пункты выполнены.

## 8. Правки контента (если попросят)
Весь текст — обычная разметка прямо в `index.html`. CSS — в теге `<style>`, JS — в `<script>` в том же файле. Отдельных css/js-файлов нет.
